# Техническое задание: ClickerCat

Кроссплатформенный автокликер с глобальной горячей клавишей и одновременной отправкой
нажатий клавиш клавиатуры и кнопок мыши.

## 1. Назначение

Простое десктопное приложение, которое по нажатию заданной пользователем горячей клавиши
эмулирует нажатие одной или нескольких клавиш клавиатуры и/или кнопок мыши. Нажатия
выполняются одновременно (одной «пачкой»), с заданной задержкой между повторами.

## 2. Технологический стек

| Параметр | Значение |
|----------|----------|
| Язык | Java 25 (LTS-уровень, `--release 25`) |
| UI | Swing (javax.swing) |
| Сборка | Maven |
| ОС | Linux, macOS, Windows |
| Артефакт | Исполняемый «fat-jar» (запуск `java -jar clickercat.jar`) |

### 2.1. Зависимости

- **JNativeHook** (`com.github.kwhat:jnativehook`) — глобальный перехват клавиатуры/мыши.
  Критически важно: стандартный Swing `KeyListener` ловит события только когда окно
  в фокусе. Для глобальной горячей клавиши (срабатывает в фоне, поверх любой игры/программы)
  нужен системный хук — это JNativeHook.
- **java.awt.Robot** (входит в JDK) — эмуляция нажатий клавиш и кнопок мыши на уровне ОС.
- **SnakeYAML** (`org.yaml:snakeyaml`) — чтение/запись конфигурации в YAML.

> Альтернатива для эмуляции — тоже JNativeHook (`GlobalScreen.postNativeEvent`), но по
> умолчанию используем `Robot` как часть JDK без доп. рисков.

## 3. Функциональные требования

### 3.1. Запуск
- Приложение запускается одним окном Swing.
- При старте загружается конфигурация из YAML (см. §5). Если файла нет — создаются значения
  по умолчанию и файл записывается.

### 3.2. Горячая клавиша срабатывания
- Пользователь задаёт **горячую клавишу** (триггер) через UI: кнопка «Назначить», после
  нажатия которой приложение перехватывает следующую нажатую пользователем клавишу и
  сохраняет её как триггер.
- Поддерживаются как клавиши клавиатуры, так и кнопки мыши в роли триггера.
- Горячая клавиша работает **глобально** — даже когда окно приложения не в фокусе.

### 3.3. Режим работы (переключатель)
Галочка/переключатель выбора режима:
- **Удержание (Hold)** — кликер активен, пока горячая клавиша зажата; отпустил —
  кликер остановился.
- **Переключение (Toggle)** — одно нажатие горячей клавиши запускает кликер,
  повторное нажатие останавливает.

### 3.4. Набор клавиш для нажатия
- Пользователь формирует **список** клавиш и кнопок, которые будут нажиматься:
  произвольная комбинация клавиш клавиатуры и кнопок мыши.
- В UI: добавление в список (кнопка «Добавить» + захват нажатия), удаление выбранного
  элемента, отображение списка.
- При срабатывании все элементы списка нажимаются **одновременно** (в рамках одной
  итерации: press всех → пауза удержания → release всех).

### 3.5. Задержка между кликами
- Поле ввода задержки между итерациями (повторами) в миллисекундах.
- Целое число ≥ 0. Значение по умолчанию — 100 мс.
- (Опционально) отдельная длительность удержания внутри одного клика (press→release),
  по умолчанию минимальная.

### 3.6. Сохранение настроек
- Все настройки сохраняются в YAML-файл в **рабочей директории** приложения
  (`./clickercat.yaml`).
- Сохранение: при изменении настроек и/или при закрытии приложения.
- Загрузка: при старте.

## 4. UI (Swing)

Одно главное окно, компактная компоновка:

```
┌─────────────────────────────────────────┐
│ ClickerCat                               │
├─────────────────────────────────────────┤
│ Горячая клавиша: [ F6 ]  [Назначить]     │
│                                          │
│ Режим:  (•) Удержание   ( ) Переключение │
│                                          │
│ Задержка между кликами: [ 100 ] мс       │
│                                          │
│ Нажимать клавиши:                        │
│  ┌────────────────────────┐              │
│  │ Mouse LEFT             │ [Добавить]   │
│  │ Key SPACE              │ [Удалить]    │
│  └────────────────────────┘              │
│                                          │
│ Статус: [ Остановлен ]    [Старт/Стоп]   │
└─────────────────────────────────────────┘
```

Требования к UI:
- Все долгие/блокирующие операции (циклы кликов, перехват) — вне Event Dispatch Thread (EDT).
  Обновление UI — только через `SwingUtilities.invokeLater`.
- Индикация состояния: активен / остановлен.

## 5. Формат конфигурации (YAML)

Файл `./clickercat.yaml`:

```yaml
hotkey:
  type: KEYBOARD        # KEYBOARD | MOUSE
  code: VC_F6           # код клавиши JNativeHook (или MOUSE_BUTTON_x)
mode: HOLD              # HOLD | TOGGLE
delayMs: 100            # задержка между итерациями кликов
holdMs: 0               # длительность удержания внутри одного клика
keys:                   # список нажимаемых клавиш/кнопок
  - type: MOUSE
    code: BUTTON1       # левая кнопка мыши
  - type: KEYBOARD
    code: VC_SPACE
```

- Невалидный/повреждённый файл → откат на значения по умолчанию + перезапись, без падения.

## 6. Логика работы кликера

1. По триггеру (с учётом режима HOLD/TOGGLE) запускается рабочий поток.
2. Цикл, пока кликер активен:
   - нажать (`press`) все элементы из `keys` одновременно;
   - выдержать `holdMs`;
   - отпустить (`release`) все элементы;
   - выдержать `delayMs`;
3. При остановке гарантированно отпустить все ранее нажатые клавиши/кнопки
   (чтобы не «залипли» при выходе/остановке).

### Маппинг кодов
- Коды JNativeHook (`NativeKeyEvent.VC_*`, `NativeMouseEvent.BUTTON*`) → коды `Robot`
  (`java.awt.event.KeyEvent.VK_*`, `InputEvent.BUTTON*_DOWN_MASK`).
- Необходим модуль-конвертер кодов.

## 7. Кроссплатформенные особенности (важно)

- **macOS**: для работы глобального хука и `Robot` требуется разрешение
  *Accessibility* (Системные настройки → Конфиденциальность и безопасность →
  Универсальный доступ). При отсутствии прав — показать понятное сообщение.
- **Linux**: JNativeHook стабильно работает на **X11**. На **Wayland** глобальный
  перехват ограничен/недоступен — задокументировать как известное ограничение,
  при невозможности инициализации хука показать сообщение.
- **Windows**: работает без доп. разрешений; антивирус/анти-чит может расценивать
  эмуляцию ввода как нежелательную активность — указать в README.

## 8. Нефункциональные требования

- Память: минимальная; одно окно, один рабочий поток кликера.
- Корректное завершение: дерегистрация хука `GlobalScreen.unregisterNativeHook()` при выходе.
- Логирование JNativeHook приглушить (по умолчанию он шумит в консоль) —
  уровень логгера выставить в `WARNING/OFF`.
- Безопасность: предусмотреть глобальную «аварийную» клавишу остановки (например `ESC`)
  на случай неконтролируемого цикла кликов.

## 9. Структура проекта (предлагаемая)

```
clickercat/
├── pom.xml
├── clickercat.yaml            # создаётся при первом запуске
└── src/main/java/cat/clicker/
    ├── App.java               # точка входа, инициализация
    ├── ui/MainWindow.java     # Swing-окно
    ├── core/Clicker.java      # рабочий поток, цикл кликов (Robot)
    ├── core/HotkeyService.java# регистрация/обработка глобального хука
    ├── config/Config.java     # POJO модели настроек
    ├── config/ConfigStore.java# загрузка/сохранение YAML
    └── input/KeyMapper.java   # маппинг кодов JNativeHook ↔ Robot
```

## 10. Сборка и запуск

- `mvn clean package` → `target/clickercat.jar` (fat-jar через `maven-shade-plugin`).
- Запуск: `java -jar clickercat.jar` (требуется установленная JRE/JDK 25).
- `maven-compiler-plugin` с `<release>25</release>`.

## 11. Критерии приёмки

1. Приложение собирается `mvn clean package` и запускается на Linux (X11), macOS, Windows.
2. Глобальная горячая клавиша срабатывает, когда окно не в фокусе.
3. Режимы HOLD и TOGGLE работают согласно §3.3.
4. Список клавиш (клавиатура + мышь) нажимается одновременно с заданной задержкой.
5. Настройки сохраняются в `./clickercat.yaml` и корректно восстанавливаются после перезапуска.
6. При остановке/выходе нет «залипших» клавиш; хук дерегистрируется.
